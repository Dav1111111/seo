## Находки

### CRITICAL

#### C1. Нет viewport meta — мобильный рендер ломается на iOS Safari
**Файл:** `frontend/app/layout.tsx`
**Что:** В `metadata` нет `viewport`. Next.js 16 раньше дефолтил viewport автоматически, но это поведение меняется и в новых версиях рекомендуется явный экспорт. Без него iOS Safari отрисует страницу шириной 980px и пользователь увидит уменьшенный десктоп.
**Доказательства:** grep по `viewport` в `frontend/app/` ничего не нашёл. В layout импортируется только `Metadata`, и поле `viewport` отсутствует.
**Фикс (5 минут):**
```ts
export const viewport: Viewport = {
  width: "device-width",
  initialScale: 1,
};
```
Проверить в браузере вручную.

#### C2. Activity feed обновляется каждые 5 секунд — ноль анонсов для скринридеров
**Файл:** `frontend/components/dashboard/activity-feed.tsx:54`, `:88-127`
**Что:** Лента поллит `/activity/current-run` каждые 5 секунд и перерисовывает список. У `<ul>` нет `aria-live`, у иконки активности — `animate-pulse` без `prefers-reduced-motion`. Скринридер NVDA/VoiceOver не получит никакого уведомления о новых событиях, для незрячего пользователя дашборд статичен. Это прямое нарушение WCAG 4.1.3 (Status Messages).
**Фикс:** на `<ul>` добавить `aria-live="polite"`, `aria-atomic="false"`, `aria-relevant="additions"`. На бейдж «работает» — `role="status"`.

#### C3. Иконочные кнопки в `priority-card` без `aria-label`
**Файл:** `frontend/components/priorities/priority-card.tsx:128-149`
**Что:** Три кнопки `<Check />`, `<Clock />`, `<X />` имеют только `title=` (Применено/Отложить/Отклонить). `title` не читается большинством скринридеров и не работает на мобиле. Без `aria-label` пользователь VoiceOver слышит «button, button, button».
**Фикс:** добавить `aria-label="Отметить применённым"`, `aria-label="Отложить"`, `aria-label="Отклонить"`. Удалить `title=` или продублировать смысл.

#### C4. Кнопка удаления конкурента — то же самое
**Файл:** `frontend/app/settings/page.tsx:240-247`
**Что:** Кнопка с `<XIcon />` имеет только `title="Удалить"`. Скринридер не озвучит контекст, к какому конкуренту относится.
**Фикс:** `aria-label={`Удалить конкурента ${d}`}`.

#### C5. Recharts грузится в основной бандл главной страницы
**Файл:** `frontend/components/dashboard/traffic-chart.tsx:6-10`, `frontend/components/dashboard/overview.tsx:14`
**Что:** `import { LineChart, Line, XAxis, ... } from "recharts"` статичный, и `TrafficChart` рендерится прямо в `OverviewPage`. Recharts со всеми зависимостями = ~150-180 KB gzipped, тащится при первой загрузке `/`. Для нетехнического владельца, который заходит с мобильника по 4G, это ощутимая задержка TTI.
**Фикс:**
```ts
const TrafficChart = dynamic(() => import("@/components/dashboard/traffic-chart").then(m => m.TrafficChart), {
  ssr: false,
  loading: () => <Skeleton className="h-64 w-full" />,
});
```

### HIGH

#### H1. Все страницы — `"use client"` на корне, RSC не используется
**Файлы:** `app/page.tsx` -> `OverviewPage` (use client), `app/competitors/page.tsx:1`, `app/priorities/page.tsx:1`, `app/reports/page.tsx:1`, `app/reports/[id]/page.tsx:1`, `app/settings/page.tsx:1`, `app/onboarding/[siteId]/page.tsx:1`.
**Что:** RootLayout — server, дальше всё клиентское. Это убивает преимущества Next.js 16 + React 19: нет streaming SSR, нет Server Actions, нет частичной гидрации, отсутствует возможность SEO-friendly рендера статичных кусков (хедеры, описания).
**Контекст:** `frontend/AGENTS.md` явно предупреждает «This is NOT the Next.js you know», но код написан в стиле Pages Router времён 13.x.
**Эффект:**
- Бандл клиента раздут — тащится `swr`, `recharts`, `@base-ui/react` для всего.
- Загрузка списка отчётов/приоритетов/отчёт по id могла бы быть SSR — данные не интерактивны до первого клика.
- Server Actions могли бы заменить `triggerRescore`, `triggerReport`, `markApplied` — без всяких proxy-route и без тащения fetch-обвязки в клиент.
**Рекомендация:** список отчётов и сам отчёт `/reports/[id]` — кандидаты на серверный рендер с `Suspense` + асинхронным `await`. Кнопки-триггеры — Server Actions.

#### H2. Polling вместо SSE/WebSocket для пайплайна
**Файлы:** `components/dashboard/activity-feed.tsx:54` (5 сек), `components/dashboard/last-run-summary.tsx:50` (4 сек), `app/competitors/page.tsx:30` (5 сек), `app/onboarding/[siteId]/page.tsx:47` (3 сек).
**Что:** Одновременно открыто 4–6 параллельных SWR-каналов с интервалами 3–10 сек. На странице `/competitors` это 5 запросов: `cp-current-run` (5s), `competitors`/`gaps`/`dive`/`opps`/`outcomes` (8s когда что-то бежит). Для backend это нагрузка, для пользователя — баттери-дрейн на мобиле.
**Альтернатива:** один SSE-канал `/sites/{id}/events` с push-обновлениями, frontend обновляет SWR-кэш через `mutate`.
**Дешёвый фикс:** объединить poll-эндпоинты в один `/sites/{id}/heartbeat`, который возвращает все актуальные срезы за один RTT.

#### H3. `traffic-chart` — нет ключа `siteId` в `useSWR`-ключе при пустом `siteId`
**Файл:** `frontend/components/dashboard/traffic-chart.tsx:13`
```ts
const { data, isLoading } = useSWR(
  `traffic-${siteId}`,            // ← если siteId="" → ключ "traffic-"
  () => api.trafficMetrics(siteId, 30),  // ← API получит "" и упадёт
  { refreshInterval: 300_000 }
);
```
В отличие от других мест, где условный ключ `siteId ? ... : null` отключает SWR, тут он всегда активен. И `api.trafficMetrics("")` сходит в `/sites//metrics/...` → `apiFetch` бросит «API call skipped: siteId is empty». Каждые 5 минут будет лишняя ошибка.
**Фикс:** `siteId ? `traffic-${siteId}` : null`.

#### H4. Типизация API ответов через `any`
**Файлы:** `frontend/lib/api.ts` целиком + несколько локальных мест.
**Что:** 38 эндпоинтов в `api`, из них около половины возвращают `any` (`dashboard`, `trafficMetrics`, `indexingMetrics`, `issues`, `agentRuns`, `sites`, `triggerPipeline`, `triggerAgent`, `queries`, `tasks` ...). На страницах потом тянется `dash?.kpis ?? {}` и т.п. без типов. Это убирает гарантии TypeScript ровно там, где они нужнее всего — на границе с backend.
**Фикс:** сгенерировать типы из FastAPI OpenAPI (есть `openapi-typescript`) или хотя бы вручную описать DashboardKpis, ReportPayload (сейчас 1 место — `report-view.tsx:7` — описано как `any` для всех полей).

#### H5. Ссылки на внешние сайты — `rel="noreferrer"` без `noopener`
**Файлы (10 мест):** `competitors/page.tsx`, `priority-card.tsx`, `last-run-summary.tsx`, `business-truth-card.tsx`, и т.д.
**Что:** Везде `target="_blank" rel="noreferrer"`. `noreferrer` сам по себе включает `noopener` в современных браузерах, но это поведение опционально, и старая Safari может пропустить. Грамотный паттерн: `rel="noopener noreferrer"`.
**Фикс:** глобальный replace `rel="noreferrer"` → `rel="noopener noreferrer"`.

#### H6. Карточки `MetricTile` — это `<a>`, обёрнутый в `cursor-pointer`, без `aria-label` для скринридера
**Файл:** `frontend/components/dashboard/last-run-summary.tsx:201-220`
**Что:** Тайл с числом «5 точек роста» — это `<Link href="/competitors">` со `<Icon><div>5</div><div>точек роста</div></Link>`. Скринридер прочитает «5 точек роста, ссылка», что нормально, но иконка `Icon` не имеет `aria-hidden`, что на VoiceOver может прочитаться как «изображение Target». Не критично, но шумно.
**Фикс:** `<Icon aria-hidden="true" />` для всех декоративных иконок (см. также H8).

#### H7. Сайдбар не сворачивается на мобиле — горизонтальный скролл гарантирован
**Файл:** `frontend/components/layout/sidebar.tsx:25` — `<aside className="w-56 shrink-0 border-r bg-muted/40 flex flex-col">`. `app-shell.tsx:50` — `<div className="flex h-full">` плюс `<main className="flex-1 overflow-auto p-6">`.
**Что:** Sidebar 224px не имеет `hidden sm:flex`, drawer-логики, бургер-меню. На экране 375px после сайдбара остаётся 151px на контент — это сломанный UX на мобиле.
**Фикс:** sidebar в drawer + бургер в шапке + хоткей `Esc` для закрытия. Использовать существующий `Dialog`/`Sheet` из shadcn (нужно добавить sheet.tsx — его нет в `components/ui/`).

#### H8. Иконки lucide без `aria-hidden` — скринридер шумит
**Что:** Большинство `<Icon className="h-4 w-4" />` рендерятся как `<svg>` без `aria-hidden`. lucide-react не выставляет `aria-hidden` по умолчанию. На VoiceOver/JAWS это значит, что для каждой кнопки и заголовка читается «изображение Activity», «изображение Brain» и т.д.
**Затронутые файлы:** все компоненты dashboard, sidebar, settings, priorities — практически везде. Точечно `aria-hidden="true"` есть только в `app-shell.tsx:16` и `site-switcher.tsx:38`.
**Фикс:** глобально — либо обернуть все иконки в `<span aria-hidden="true">`, либо завести wrapper-компонент `<Icon decorative />` , который автоматически прокидывает `aria-hidden`.

### LOW

#### L1. Типографика на пределе читаемости
**Файлы:** widespread — `text-[10px]`, `text-[11px]` встречается 19+ раз.
**Что:** Бейджи, счётчики, `text-muted-foreground` — всё это размером 10–11px на классе `text-muted-foreground` (это `oklch(0.556 0 0)` = средне-серый). Контраст 10px серого на белом фоне ниже WCAG AA для нормального текста (требуется 4.5:1; для крупного — 3:1, но крупным считается ≥18px или ≥14px bold).
**Не сканировал контраст пиксельно**, но классы `text-[10px] text-muted-foreground` и `text-[11px] text-muted-foreground` — почти гарантированно ниже 4.5:1. Проверить в DevTools или axe DevTools.
**Фикс (квик-вин):** на класс `text-muted-foreground` поднять до `oklch(0.45 0 0)` (тёмнее) для случаев <12px. Лучше — увеличить кегль до 12px (`text-xs`) минимум.

#### L3. Магические цвета `bg-emerald-100/text-emerald-800/border-emerald-300` повторяются

**WCAG 2.4.1.**
**Фикс:** 5 строк в layout + один CSS-класс.

#### L8. Print-стиль игнорирует light/dark
**Файл:** `frontend/app/globals.css:111-117`
**Что:** При печати `body { background: white }`, но в dark-режиме текст уже белый. Получится белый-на-белом.

**Файл:** `frontend/app/error.tsx:14-19`
**Что:** Не критично, но нарушает консистентность. Ничего не ломается.

## Доступность (WCAG 2.2 AA — критерии)

### 1.1 Text alternatives — **Fail**
- 1.1.1 Non-text Content: иконки lucide без `aria-hidden` (H8). Чарт Recharts без альтернативы (M10). Иконочные кнопки без `aria-label` (C3, C4).

- 1.3.1 Info and Relationships: form inputs без явных `<label>` (M2). Settings radios без `role="radiogroup"` (M1). Лента активности без `role="log"` или `aria-live` (C2). Семантика заголовков OK — `<h1>` на каждой странице, но `<h2>` пропускается (сразу `<h3>` или вообще `<CardTitle>` который рендерится как `<div>`).
- 1.3.2 Meaningful Sequence: OK.

### 1.4 Distinguishable — **Fail**
- 1.4.3 Contrast: `text-[10px] text-muted-foreground`, `text-[11px] text-muted-foreground` — подозрительно низкий контраст (L1). На `business-truth-card.tsx:319` Badge `text-[10px] font-mono` поверх `bg-rose-50/text-rose-700` — проверить в axe.
- 1.4.4 Resize text: используется `rem`/Tailwind единицы — OK на zoom 200%. Проверить вручную.

- 1.4.11 Non-text Contrast: бейджи priority с тоном `bg-slate-100 text-slate-700` — подозрительно. Кнопка переключения сайтов — мелкая (h-3 w-3 для chevron).
- 1.4.12 Text Spacing: `leading-snug`, `leading-tight` — может ломать, проверить вручную.
- 1.4.13 Content on Hover or Focus: всплывашки `title="..."` (priority-card) — нельзя dismiss клавиатурой.

### 2.1 Keyboard accessible — **Partial Pass**
