## ПРОБЛЕМА

### SEO-оценка сайта: 8 из 100

Сайт **фактически невидим для поисковых систем**. Причина — архитектурная: весь контент рендерится JavaScript'ом в браузере, а поисковые роботы видят пустую HTML-страницу.

### Детальная диагностика:

| Проблема | Критичность | Описание |
|----------|-------------|----------|
| Пустой HTML | КРИТИЧЕСКАЯ | `<body>` содержит только `<div id="root"></div>`. Весь контент (тексты, цены, описания экспедиций) — внутри JavaScript. Яндекс не рендерит JS надёжно. |
| Одинаковый title | КРИТИЧЕСКАЯ | На ВСЕХ страницах один и тот же `<title>Grand Tour Spirit</title>`. Яндекс не может различить страницы между собой. |
| Нет meta description | КРИТИЧЕСКАЯ | Ни на одной странице нет описания для поисковой выдачи. Яндекс показывает пустой или случайный сниппет. |
| robots.txt отдаёт HTML | КРИТИЧЕСКАЯ | SPA перехватывает запрос к `/robots.txt` и отдаёт HTML-страницу вместо текстового файла. Поисковик не может прочитать правила сканирования. |
| sitemap.xml отдаёт HTML | КРИТИЧЕСКАЯ | Аналогично — поисковик не может обнаружить список страниц сайта. |
| Нет canonical URL | ВЫСОКАЯ | Нет указания на каноническую версию страницы. www и non-www версии дублируют друг друга. |
| non-www не редиректит | ВЫСОКАЯ | `grandtourspirit.ru` и `www.grandtourspirit.ru` отдают одинаковый контент — полное дублирование сайта. |
| Soft 404 | ВЫСОКАЯ | Любой несуществующий URL (например `/asdfgh`) отдаёт HTTP 200. Бесконечное пространство "страниц" для поисковика. |
| Нет OG-тегов | ВЫСОКАЯ | При шаринге в Telegram, VK, WhatsApp — пустая карточка без превью. Для премиум-бренда это критично. |
| `lang="en"` | СРЕДНЯЯ | Указан английский язык вместо русского. Вводит поисковики в заблуждение. |
| Нет Schema.org | ВЫСОКАЯ | Нет структурированных данных. Яндекс не может показать цены, рейтинги, тип бизнеса в выдаче. |
| manifest.json | СРЕДНЯЯ | Указано "My App" вместо "Grand Tour Spirit". |
| Core Web Vitals | ВЫСОКАЯ | LCP (время до отображения контента) оценочно 3-5 секунд на мобильных из-за цепочки загрузки JS. |

### Что это значит для бизнеса:

- Сайт не находится по запросам "багги Сочи", "экспедиции Абхазия", "премиум туры Сочи"
- Потенциальные клиенты ищут эти запросы, но находят конкурентов
- Блог со статьями существует, но поисковики не видят его контент
- Деньги на контент и разработку сайта не конвертируются в органический трафик

## ПЛАН РЕАЛИЗАЦИИ

### Фаза 1: Аварийные исправления nginx (1-2 дня)

**Цель**: Дать поисковикам базовую информацию о сайте. Не требует изменений в React-коде.

**Что делать:**

#### 1.1. Физический robots.txt

Создать файл `/var/www/grandtourspirit/static/robots.txt`:
```
User-agent: *
Allow: /
Disallow: /admin/
Disallow: /login
Disallow: /portal

Sitemap: https://www.grandtourspirit.ru/sitemap.xml
```

В nginx добавить ДО блока `try_files`:
```nginx
location = /robots.txt {
    alias /var/www/grandtourspirit/static/robots.txt;
    default_type text/plain;
}
```

#### 1.2. Физический sitemap.xml

Создать файл `/var/www/grandtourspirit/static/sitemap.xml`:
```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://www.grandtourspirit.ru/</loc>
    <changefreq>weekly</changefreq>
    <priority>1.0</priority>
  </url>
  <url>
    <loc>https://www.grandtourspirit.ru/abkhazia</loc>
    <changefreq>weekly</changefreq>
    <priority>0.9</priority>
  </url>
  <url>
    <loc>https://www.grandtourspirit.ru/experiences</loc>
    <changefreq>monthly</changefreq>
    <priority>0.8</priority>
  </url>
  <url>
    <loc>https://www.grandtourspirit.ru/stories</loc>
    <changefreq>daily</changefreq>
    <priority>0.7</priority>
  </url>
  <url>
    <loc>https://www.grandtourspirit.ru/contacts</loc>
    <changefreq>monthly</changefreq>
    <priority>0.5</priority>
  </url>
  <url>
    <loc>https://www.grandtourspirit.ru/partners</loc>
    <changefreq>monthly</changefreq>
    <priority>0.5</priority>
  </url>
</urlset>
```

В nginx:
```nginx
location = /sitemap.xml {
    alias /var/www/grandtourspirit/static/sitemap.xml;
    default_type application/xml;
}
```

#### 1.3. Редирект non-www → www

В nginx добавить отдельный server-блок:
```nginx
server {
    listen 443 ssl;
    server_name grandtourspirit.ru;
    # SSL сертификат...
    return 301 https://www.grandtourspirit.ru$request_uri;
}
```

#### 1.4. Редирект trailing slash

В основном server-блоке:
```nginx
rewrite ^(.+)/$ $1 permanent;
```

#### 1.5. Исправить lang и manifest

В `index.html`:
- Заменить `<html lang="en">` на `<html lang="ru">`

В `manifest.json`:
- Заменить `"name": "My App"` на `"name": "Grand Tour Spirit"`
- Заменить `"short_name": "My App"` на `"short_name": "GTS"`
- Заменить `"description": "My App description"` на `"description": "Премиальный клуб активного отдыха в Сочи"`

**Результат фазы 1**: Поисковики находят robots.txt и sitemap.xml, знают какие страницы существуют. Дублирование www/non-www устранено. Но контент страниц всё ещё невидим.

### Фаза 2: Мета-теги через nginx sub_filter (3-5 дней)

**Цель**: Дать каждой странице уникальный title, description, OG-теги и Schema.org разметку. Без изменений в React.

**Как работает**: nginx модуль `sub_filter` подменяет части HTML на лету, в зависимости от запрошенного URL.

**Что делать:**

#### 2.1. Включить sub_filter в nginx

```nginx
sub_filter_once on;
sub_filter_types text/html;
```

#### 2.2. Мета-теги для каждой страницы

### Фаза 3: SSR / Pre-rendering (2-4 недели)

**Цель**: Сделать весь контент видимым для поисковиков. Это корневое решение проблемы.

**Проблема, которую решаем**: Фазы 1-2 дают поисковикам title и description, но основной контент страниц (тексты экспедиций, цены, программы, блог) по-прежнему внутри JavaScript. Яндекс может его не увидеть.

#### Рекомендуемый подход: Vike (vite-plugin-ssr)

**Почему Vike**: сохраняет текущий стек (Vite + React), добавляет server-side rendering с минимальными изменениями.

**Что нужно сделать:**

1. **Установить Vike** и настроить SSR-pipeline
2. **Заменить кастомный state-роутер** на file-based routing Vike (самая большая часть работы)
3. **Добавить `react-helmet-async`** для динамического управления head-тегами из React-компонентов
4. **Каждый компонент страницы** должен экспортировать свои meta-теги:
   ```jsx
   export function AbkhaziaPage() {
     return (
       <>
         <Helmet>
           <title>Багги-экспедиции по Абхазии — от 24 900₽ | GTS</title>
           <meta name="description" content="..." />
         </Helmet>
         {/* ... контент страницы ... */}
       </>
     );
   }
   ```
5. **SSR для /stories**: блог-статьи из Supabase рендерятся на сервере — Яндекс видит полный текст каждой статьи
6. **Настроить Node.js сервер** (или serverless) для SSR вместо чистого nginx для статики

**Результат фазы 3**: Весь контент всех страниц виден поисковикам. Core Web Vitals улучшаются (LCP с 3-5с до <1.5с). SEO-оценка: с 35 до 75-85/100.
